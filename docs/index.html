Visão Geral

O Sistema Quatro Cantos é uma solução completa de gestão empresarial desenvolvida em Python, projetada para gerenciar operações, estoque, finanças e recursos humanos de forma integrada e eficiente.

OP

Módulo Operacional

Cálculo de capacidade produtiva baseado em turnos de trabalho, análise de ociosidade e percentual de utilização da fábrica.

ES

Gestão de Estoque

Controle completo de entrada e saída de produtos com rastreabilidade, validação de quantidades e integração com banco de dados.

FN

Módulo Financeiro

Análise de custos, precificação com margem de lucro, projeções de faturamento e cálculo de indicadores como ROI e ponto de equilíbrio.

RH

Recursos Humanos

Cálculo de folha de pagamento com INSS e IR progressivos, gestão de horas extras e hierarquia de cargos.

Importante: Este sistema foi desenvolvido com foco em boas práticas de programação, incluindo separação de lógica de negócio, validação de dados e documentação completa do código.

Instalação e Configuração

Clonar o Repositório

Faça o download do código fonte do GitHub para o seu computador.

git clone https://github.com/gabrielamnss1/Quatro-Cantos.git cd Quatro-Cantos

Criar Ambiente Virtual

Crie um ambiente virtual Python isolado para o projeto.

python -m venv venv

Ativar o Ambiente Virtual

Ative o ambiente virtual de acordo com seu sistema operacional.

# Windows venv\Scripts\activate # Linux/Mac source venv/bin/activate

Instalar Dependências

Instale todas as bibliotecas necessárias usando o pip.

pip install sqlalchemy python-dotenv bcrypt

Configurar Banco de Dados (Opcional)

Crie um arquivo .env na raiz do projeto para configurar o banco de dados.

# Arquivo .env DATABASE_URL=sqlite:///dados.db # Para PostgreSQL # DATABASE_URL=postgresql://usuario:senha@localhost/nome_banco
Nota: Se o arquivo .env não for criado, o sistema usará SQLite automaticamente.

Executar o Sistema

Inicie o sistema executando o arquivo principal.

python main.py

Módulos do Sistema

1. Módulo Operacional

Arquivo: operacional.py

Calcula a capacidade produtiva da fábrica com base no número de turnos ativos (Manhã, Tarde e/ou Noite).

Funcionalidade Descrição
Cálculo de Capacidade 1.666 unidades por turno, multiplicado pelo número de turnos ativos
Projeções Capacidade diária, mensal (30 dias) e anual (12 meses)
Análise de Ociosidade Calcula unidades ociosas e percentual de uso da capacidade máxima
Validação Aceita apenas valores entre 1 e 3 turnos
Exemplo de Uso: Com 2 turnos ativos, a capacidade será de 3.332 unidades/dia, 99.960 unidades/mês e 1.199.520 unidades/ano, representando 66,67% da capacidade máxima.

2. Gestão de Estoque

2.1 Entrada de Produtos

Arquivo: estoque_entrada.py

Registra a entrada de novos produtos no estoque com validação completa.

Campo Tipo Validação
Nome do Produto String Mínimo 3 caracteres
Quantidade Integer Maior que zero
Preço Unitário Float Maior que zero

2.2 Saída de Produtos

Arquivo: estoque_saida.py

Registra vendas e saídas de produtos com verificação de disponibilidade.

Atenção: O sistema verifica se há estoque suficiente antes de permitir a saída. Tentativas de retirar mais do que o disponível resultarão em erro.

3. Módulo Financeiro

Arquivo: financeiro.py

Realiza análise financeira completa com cálculo de custos, precificação e projeções de lucro.

Custos Considerados

  • Conta de Água
  • Conta de Luz (Energia Elétrica)
  • Impostos Gerais (Taxas, alvarás, etc.)
  • Folha de Pagamento Completa

Indicadores Calculados

Indicador Descrição
Custo por Unidade Custo total dividido pelo volume de movimentação
Preço de Venda Custo unitário + 50% de margem de lucro
ROI (Return on Investment) Retorno sobre o investimento em percentual
Ponto de Equilíbrio Quantidade necessária para cobrir custos
Projeção Anual Receita e lucro projetados para 12 meses

4. Módulo de Recursos Humanos

Arquivo: rh.py

Calcula a folha de pagamento com descontos de INSS e IR de acordo com as tabelas progressivas de 2025.

Hierarquia de Cargos

Cargo Valor/Hora Horas Extras
Operário R$ 15,00 Sim (2x)
Supervisor R$ 40,00 Sim (2x)
Gerente R$ 60,00 Não
Diretor R$ 80,00 Não

Tabela INSS 2025 (Progressiva)

Faixa Salarial Alíquota
Até R$ 1.412,00 7,5%
R$ 1.412,01 a R$ 2.666,68 9%
R$ 2.666,69 a R$ 4.000,03 12%
Acima de R$ 4.000,04 14% (teto: R$ 908,85)

Tabela IR 2025 (Progressiva)

Faixa Salarial Alíquota Parcela a Deduzir
Até R$ 2.259,20 Isento -
R$ 2.259,21 a R$ 2.826,65 7,5% R$ 169,44
R$ 2.826,66 a R$ 3.751,05 15% R$ 381,44
R$ 3.751,06 a R$ 4.664,68 22,5% R$ 662,77
Acima de R$ 4.664,68 27,5% R$ 896,00
Observação: O cálculo do IR é feito sobre a base de cálculo (salário bruto - INSS). Horas extras são pagas com valor dobrado (2x o valor da hora normal).

Banco de Dados

Arquivo: database.py

O sistema utiliza SQLAlchemy ORM (Object-Relational Mapping) para gerenciar o banco de dados, permitindo trabalhar com dados usando objetos Python ao invés de SQL puro.

Estrutura das Tabelas

Tabela: produtos

Campo Tipo Descrição
id Integer (PK) Chave primária, auto-incremento
nome String(100) Nome do produto
quantidade Integer Quantidade em estoque
preco Float Preço unitário do produto

Tabela: funcionarios

Campo Tipo Descrição
id Integer (PK) Chave primária, auto-incremento
nome String(100) Nome completo do funcionário
cargo String(50) Cargo: Operário, Supervisor, Gerente ou Diretor
salario Float Salário líquido do funcionário

Configuração do Banco

O banco de dados é configurado através de variável de ambiente, permitindo usar diferentes SGBDs:

# SQLite (padrão - arquivo local) DATABASE_URL=sqlite:///dados.db # PostgreSQL (produção) DATABASE_URL=postgresql://usuario:senha@localhost:5432/nome_banco # MySQL DATABASE_URL=mysql://usuario:senha@localhost:3306/nome_banco
Vantagem do ORM: O SQLAlchemy permite trocar de banco de dados alterando apenas a URL de conexão, sem precisar modificar nenhum código do sistema.

Como Usar o Sistema

Pré-requisito: Certifique-se de que o ambiente virtual está ativado e todas as dependências foram instaladas antes de executar o sistema.

Iniciar o Sistema

Execute o arquivo principal do sistema.

python main.py

Menu Principal

O sistema exibirá um menu com todas as opções disponíveis:

================================================== SISTEMA DE GESTAO - QUATRO CANTOS ================================================== [1] Modulo Operacional - Capacidade Produtiva [2] Estoque - Entrada de Produtos [3] Estoque - Saida de Produtos [4] Modulo Financeiro - Custos e Lucros [5] RH - Folha de Pagamento [0] Sair do Sistema ==================================================

Selecionar Módulo

Digite o número correspondente ao módulo desejado e pressione Enter.

Dica: Digite apenas o número da opção. Valores inválidos exibirão uma mensagem de erro e retornarão ao menu principal.

Inserir Dados

Cada módulo solicitará diferentes informações. Siga as instruções na tela e digite os valores quando solicitado.

  • Números inteiros: Digite apenas números sem casas decimais (ex: 3)
  • Valores monetários: Use ponto como separador decimal (ex: 1500.50)
  • Texto: Digite normalmente, respeitando o mínimo de caracteres quando especificado

Visualizar Resultados

O sistema processará os dados e exibirá os resultados formatados na tela, incluindo tabelas, totalizações e análises.

Retornar ao Menu

Após visualizar os resultados, pressione Enter para retornar ao menu principal e selecionar outro módulo ou sair do sistema.

Exemplos de Uso

Exemplo 1: Calcular Capacidade Produtiva

  1. Execute o sistema: python main.py
  2. Digite 1 para acessar o Módulo Operacional
  3. Quando solicitado, digite o número de turnos (ex: 2)
  4. O sistema exibirá:
    • Capacidade diária: 3.332 unidades
    • Capacidade mensal: 99.960 unidades
    • Capacidade anual: 1.199.520 unidades
    • Percentual de uso: 66,67%
    • Unidades ociosas: 1.666 por dia
  5. Pressione Enter para retornar ao menu

Exemplo 2: Registrar Entrada de Produto

  1. No menu principal, digite 2 para Estoque - Entrada
  2. Nome do produto: Tijolo Refratário
  3. Quantidade: 1000
  4. Preço unitário: 5.50
  5. O sistema salvará no banco de dados e confirmará o registro
  6. Pressione Enter para retornar ao menu

Exemplo 3: Calcular Folha de Pagamento

  1. No menu principal, digite 5 para RH - Folha de Pagamento
  2. Número de funcionários: 3
  3. Para cada funcionário, informe:
    • Nome: João Silva
    • Cargo: Operário
    • Horas extras: 10
  4. O sistema calculará automaticamente:
    • Salário bruto (160h × valor/hora + horas extras)
    • Desconto INSS (tabela progressiva)
    • Desconto IR (tabela progressiva)
    • Salário líquido
  5. Exibirá relatório completo com todos os funcionários e totalizações

Tecnologias Utilizadas

Python 3.x

Linguagem

Linguagem de programação principal do projeto, escolhida por sua simplicidade, legibilidade e vasto ecossistema de bibliotecas.

SQLAlchemy

ORM

Biblioteca ORM (Object-Relational Mapping) que permite trabalhar com bancos de dados usando objetos Python, abstraindo SQL complexo.

Gerencia variáveis de ambiente através de arquivo .env, permitindo configurar o sistema sem alterar código.

Biblioteca para hash seguro de senhas, utilizando salt automático e algoritmo resistente a ataques de força bruta.

Boas Práticas: O código segue PEP 8 (guia de estilo Python), utiliza nomes descritivos para variáveis e funções, e possui documentação inline extensiva para facilitar manutenção e aprendizado.

Segurança e Boas Práticas

Hash de Senhas

Utiliza bcrypt para armazenar senhas de forma segura, com salt automático e algoritmo lento para prevenir ataques de força bruta.

Validação de Entrada

Todas as entradas do usuário são validadas quanto a tipo, range e formato antes de serem processadas ou armazenadas no banco.

Variáveis de Ambiente

Credenciais e configurações sensíveis são armazenadas em arquivo .env, que não é versionado no Git (.gitignore).

ORM (Proteção contra SQL Injection)

Uso de SQLAlchemy ORM previne ataques de SQL injection, pois as queries são parametrizadas automaticamente.

Importante: Nunca compartilhe o arquivo .env com credenciais reais. Sempre use senhas fortes e rotacione-as periodicamente em ambiente de produção.
